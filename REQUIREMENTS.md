# Requirements

## 1. Authentication & Roles

- JWT-based auth via Supabase Auth (email + password)
- Role system: `admin`, `user` (roles stored in a `profiles` table linked to `auth.users`)
- An **admin seed account** is created on first deploy (credentials defined in server env vars)
- Admin capabilities to be defined in a later phase

## 2. Onboarding Flow

Triggered on first login when the user has no profile data saved.

### Questions to collect:

**Body metrics**
- Age
- Biological sex (Male / Female)
- Height — user chooses cm or ft/in at start of onboarding; preference saved to profile
- Weight — user chooses kg or lbs; preference saved to profile
- Estimated body fat % (optional, with a visual guide to help estimate)

**Fitness background**
- How many days per week do you currently work out? (0–7)
- How long have you been training? (Beginner < 1yr / Intermediate 1–3yr / Advanced 3yr+)
- Do you have access to a gym, home equipment, or no equipment?

**Goal**
- Lose weight
- Gain muscle
- Lose weight & gain muscle (body recomposition)
- Maintain current physique
- Improve general fitness / endurance

**Goal weight** *(collected when goal is lose_weight, gain_muscle, or recomp)*
- Target weight input (respects the user's chosen unit — kg or lbs)
- Immediately after entry, show a **timeline projection table**: how long it would take to reach goal weight at each caloric adjustment level from ±400 to ±1000 kcal/day in 100 kcal steps
- Direction is automatic: deficit rows for weight loss goals, surplus rows for muscle gain; recomp shows both
- Calculation basis: 1 kg of tissue ≈ 7,700 kcal (standard energy balance model)
- Display format: each row shows the daily adjustment and the estimated weeks/months to goal (e.g. "−500 kcal/day → ~10 weeks")
- A note is shown that faster rates (>1% bodyweight/week loss, >0.5 kg/week gain) carry higher risk of muscle loss or excess fat gain respectively
- Goal weight and the user's chosen daily deficit/surplus are saved to the profile for use in dashboard calorie targets

**Lifestyle**
- Activity level outside of workouts (sedentary / lightly active / moderately active / very active)
- Any injuries or limitations? (free text, optional)
- Dietary preference (no restriction / vegetarian / vegan / other) — for future calorie/macro features

All collected data is saved to the user's `profiles` row in the database.

## 3. AI Personal Trainer Agent

Activated after onboarding is complete. Acts as a licensed, evidence-based personal trainer grounded in current fitness literature (periodisation, progressive overload, energy balance, etc.).

### Behaviour
- Analyses the user's profile (goal, metrics, training history, equipment access)
- Generates an initial personalised assessment summary
- Answers follow-up questions in a conversational chat interface
- Recommendations are always accompanied by brief scientific rationale
- Does not provide medical advice; includes a disclaimer for users with health conditions

### Constraints
- Powered by Anthropic Claude API (`claude-sonnet-4-6`)
- System prompt encodes the "personal trainer" persona and all user profile data at session start
- Conversation history is persisted per user in the database

## 4. Core Feature Outputs

### 4.1 Workout Split
Generated by the AI agent based on:
- Weekly training frequency
- Goal (hypertrophy / fat loss / recomp / endurance)
- Training level (beginner / intermediate / advanced)
- Available equipment
- Age (recovery capacity consideration)

Example splits the agent may recommend: Full Body, Upper/Lower, Push/Pull/Legs, Body Part Split, etc.

The recommended split is saved as the user's active programme and shown on the dashboard.

### 4.2 Calorie Targets
Calculated using standard formulas (Mifflin-St Jeor for BMR, activity multiplier for TDEE):

- **Maintenance calories** (TDEE)
- **Active target** — user's chosen daily surplus/deficit (selected during onboarding via the goal weight timeline table)
- **Macro breakdown** — protein / carbs / fat targets (g and %)

Displayed on dashboard and updated whenever the user updates their profile metrics.

## 5. Workout Tracking

Accessible via the **Workouts** section. First-time users go through a 2-step setup flow before reaching the schedule view.

### 5.1 Split Selection (first-time screen)

Cards are generated based on the user's `workout_days_per_week` profile value. Each card shows:
- Split name
- Description paragraph
- One **Pro** and one **Con** vs other splits for that frequency
- If only one split fits the frequency, pro/con are omitted

Available splits per frequency:

| Days/week | Available splits |
|---|---|
| 1 | Full Body |
| 2 | Full Body ×2, Upper/Lower |
| 3 | Full Body ×3, Push/Pull/Legs, Upper/Lower/Full |
| 4 | Upper/Lower ×2, PPL+, 4-Day Body Part |
| 5 | PPL + Upper/Lower, 5-Day Body Part |
| 6–7 | Push/Pull/Legs ×2, 6-Day Body Part |

Chosen split is saved to the user's profile.

### 5.2 Exercise Selection (second-time screen)

Shown after split selection. Organised by the muscle groups present in the chosen split. For each muscle group:
- A list of popular exercises is shown
- Clicking an exercise opens a detail panel showing:
  - Short description (2–3 sentences)
  - Animated visual of the movement
  - Toggle to add/remove from the user's list

Chosen exercises are saved per user.

### 5.3 Workout Schedule (main view)

After setup, shows the week laid out day by day based on the chosen split. Each day card displays:
- Muscle groups targeted that day
- List of the user's chosen exercises for those muscle groups
- Clicking a day expands it to show all exercises with sets/reps guidance

Users can reconfigure their split or exercises at any time via a button on the schedule screen.

## 6. Out of Scope (for now)
